OverByte

Reverse compression. Make files bigger.

Instead of reducing file size, OverByte increases it — while preserving the original file format and keeping the file fully functional. Inflated files still open and behave normally in standard applications.

For example

A 25KB JPEG can be expanded to 10GB.
The resulting JPEG still opens normally.
No transcoding, re-encoding, or format conversion occurs.
The original file data remains intact.

Why?

Stress testing applications that handle large files

Testing upload size limits

Observing file system behavior at extreme sizes

Analyzing decoder and parser behavior

Simulating large datasets without downloading real ones

Details
  • Preserves the file's original structure
  • Appends valid padding that doesn't interfere with decoders
  • Does not modify the original content data
  • Does not re-encode media
  • Does not convert formats
  • Does not break file integrity
  • Developers
  • Researchers
  • Security testers
  • File system experimentation

Warning

  • Inflating files to very large sizes can consume significant disk space
  • May impact system performance
  • May trigger antivirus or security warnings due to unusual behavior
  • Always test in a controlled environment
